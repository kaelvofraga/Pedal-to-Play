Regras de Negócio

Módulo Manter Usuário

1.01 = Email deverá estar no formato padrão de email. Caso contrário exibi-se a MSG 1.01.

1.02 = Senha deve conter de 6 a 30 caracteres. Caso contrário exibi-se a MSG 1.02.

1.03 = Para o formulário de cadastro ser válido também é necessário que a senha e a senha de confirmação sejam idênticas. Caso contrário exibi-se a MSG 1.03.

1.04 = Os campos de email, senha e confirmação de senha são obrigatórios, caso não algum não seja preenchido deve-se exibir a mensagem MSG 1.04.

1.05 = Não deve-se habilitar o registro de um email já registrado. Caso contrário exibi-se a MSG 1.05.

1.06 = Ao registrar um novo usuário deve ser salva a data de inscrição deste.

Módulo Avatar

2.01 = O avatar é composto por dois elementos: o ciclista e a bicicleta.

2.02 = O ciclista é dividido em 3 partes interligadas: cabelo, rosto (olhos, boca e nariz) e equipamentos.

2.03 = Os equipamentos incluem: capacete, trage, calçados e proteções (luva, joelheira e cotoveleira, óculos).

2.04 = Cada parte do ciclista tem 3 a 5 variações disponíveis e para cada variação pode-se escolher uma cor.

2.05 = Cada desafio cumprido deve presentear o usuário com alguma recompensa.

2.06 = Cada nível torna disponível ao usuário um novo modelo de bicicleta e uma parte da vestimenta profissional.

2.07 = Os modelos, ordenados por nível, são os seguintes:

  1. Cruiser
  2. BMX
  3. Moutain
  4. Road
  5. Triathlon

2.08 = As vestimentas, ordenados por nível, são os seguintes:

  1. Cabacete
  2. Luvas
  3. Tênis
  4. Óculos
  5. Traje

2.09 = Quando uma customização do avatar for salva mas não haver conexão com o servidor, esta pendência deve ser adicionada a uma fila no armazenamento local para ser salva posteriormente.

2.10 = Quando o usuário deseja trocar o gênero do avatar, o sistema exbibe a mensagem de confirmação MSG 2.01.

2.11 = Quando ocorre uma falha no salvamento do avatar, no servidor, o sistema exibe a mensagem MSG 2.02.

2.12 = Quando ocorre uma falha ao recuperar as customizações salvas do avatar, no servidor, o sistema exibe a mensagem MSG 2.03.

2.13 = Quando o dispositivo do usuário não suporta SVG inline, o sistema exibe a mensagem MSG 2.04.


Mensagens do Sistema

-->Opções para tipo de mensagem, utilizados ao final entre colchetes [Notificação, Alerta, Erro ou Confirmação]

1.01 = [Erro] Email inválido.

1.02 = [Erro] Senha inválida. Deve conter de 6 a 30 caracteres.

1.03 = [Erro] As senhas digitadas são diferentes.

1.04 = [Erro] Este campo deve ser preenchido.

1.05 = [Erro] Este email já está sendo utilizado.


2.01 = [Pergunta] Deseja trocar o gênero de seu avatar?

2.02 = [Erro] Customizações de avatar inválidas. Confira suas escolhas.

2.03 = [Erro] A customização de seu avatar não pode ser recuperada.

2.04 = [Erro] Desculpe-me, mas seu dispositivo não suporta o sistema de customização do avatar.
