Regras de Negócio

Módulo Manter Usuário

1.01 = Email deverá estar no formato padrão de email. Caso contrário exibi-se a MSG 1.01.

1.02 = Senha deve conter de 6 a 30 caracteres. Caso contrário exibi-se a MSG 1.02.

1.03 = Para o formulário de cadastro ser válido também é necessário que a senha e a senha de confirmação sejam idênticas. Caso contrário exibi-se a MSG 1.03.

1.04 = Os campos de email, senha e confirmação de senha são obrigatórios, caso não algum não seja preenchido deve-se exibir a mensagem MSG 1.04.

1.05 = Não deve-se habilitar o registro de um email já registrado. Caso contrário exibi-se a MSG 1.05.

1.06 = Ao registrar um novo usuário deve ser salva a data de inscrição deste.

Módulo Monitoramento de Pedalada

2.01 = Os dados da pedalada são organizados em um arranjo de objetos do tipo Position
(retorno do plugin Geolocation do Cordova ao obter a localização do usuário com sucesso),
esse objeto deve conter os seguintes atributos com os devidos tipos listados abaixo:

  1. coords: arranjo de coordenadas. (Coordinates)
    • latitude: Latitude em graus decimais. (Number)
    • longitude: Longitude em graus decimais. (Number)
    • altitude: Altura da posição em metros. (Number)
    • accuracy: Nível de precisão da latitude e longitude em metros. (Number)
    • altitudeAccuracy: Nível de precisão da altitude em metros. (Number)
    • heading: Direcção da viagem, especificado em graus contando no sentido horário em relação ao norte verdadeiro. (Number)
    • speed: A velocidade atual do aparelho em relação ao chão, especificado em metros por segundo. (Number)
  2. timestamp: timestamp do sistema. (DOMTimeStamp)

2.02 = Caso os dados a serem salvos da pedalada não atendam a regra RNG 2.01, exibi-se a MSG 2.01.

2.03 = Caso o GPS não esteja ativo, exibi-se a MSG 2.02

Módulo Avatar

3.01 = O avatar é composto por dois elementos: o ciclista e a bicicleta.

3.02 = O ciclista é dividido em 3 partes interligadas: cabelo, rosto (olhos, boca e nariz) e equipamentos.

3.03 = Os equipamentos incluem: capacete, trage, calçados e proteções (luva, joelheira e cotoveleira, óculos).

3.04 = Cada parte do ciclista tem 3 a 5 variações disponíveis e para cada variação pode-se escolher uma cor.

3.05 = Cada desafio cumprido deve presentear o usuário com alguma recompensa.

3.06 = Cada nível torna disponível ao usuário um novo modelo de bicicleta e uma parte da vestimenta profissional.

3.07 = Os modelos, ordenados por nível, são os seguintes:

  1. Cruiser
  2. BMX
  3. Moutain
  4. Road
  5. Triathlon

3.08 = As vestimentas, ordenados por nível, são os seguintes:

  1. Cabacete
  2. Luvas
  3. Tênis
  4. Óculos
  5. Traje

3.09 = Quando uma customização do avatar for salva mas não haver conexão com o servidor, esta pendência deve ser adicionada a uma fila no armazenamento local para ser salva posteriormente.

3.10 = Quando o usuário deseja trocar o gênero do avatar, o sistema exbibe a mensagem de confirmação MSG 2.01.

3.11 = Quando ocorre uma falha no salvamento do avatar, no servidor, o sistema exibe a mensagem MSG 2.02.

3.12 = Quando ocorre uma falha ao recuperar as customizações salvas do avatar, no servidor, o sistema exibe a mensagem MSG 2.03.

3.13 = Quando o dispositivo do usuário não suporta SVG inline, o sistema exibe a mensagem MSG 2.04.
